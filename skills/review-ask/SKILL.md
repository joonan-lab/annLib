# /review-ask

질문에 답하고 더 깊은 질문을 생성합니다. **반복할수록 이해가 깊어집니다.**

## 실행 절차

### 1. 질문 확인
ARGUMENTS 를 질문으로 사용합니다. 없으면 question-tree.md 를 보여주고 고르게 합니다.

### 2. 현재 레벨 판단
question-tree.md 에서 이 질문이 L1인지 L2인지 판단합니다.
- 처음 나온 질문이면 L1 질문에서 나온 것으로 간주
- 없으면 현재 가장 많이 채워진 레벨 + 1

다음 레벨 = 현재 레벨 + 1 (최대 L4)

### 3. 관련 논문 파악
papers/ 의 모든 .md 파일을 Read 도구로 읽습니다.
질문과 가장 관련된 논문을 2-5편 선별합니다.

### 4. 답변 생성
선별한 논문들을 근거로 답변합니다.

**답변 구조:**
```markdown
## 답변

{2-4문단의 답변. 각 주장마다 [저자 et al. (연도)] 형식으로 인용}

## 근거 논문
{답변에 사용한 논문 목록 — citation_str 형식}
```

**좋은 답변의 기준:**
- 논문 간 일치하는 부분과 충돌하는 부분을 모두 언급
- "논문 X에 따르면"이 아니라 내용 중심으로 서술
- 확실한 것과 불확실한 것을 구분

### 5. 다음 레벨 질문 생성 (핵심)
답변을 바탕으로 **다음 레벨 질문 3개**를 생성합니다.

레벨별 질문 특성:
- **L2로 갈 때**: 여러 논문을 통합해야 답할 수 있는 질문
- **L3로 갈 때**: 논문들이 공통으로 전제하는 것에 의문을 품는 질문
- **L4로 갈 때**: 현재 문헌으로는 답할 수 없는, 새로운 연구가 필요한 질문

각 질문은:
1. 방금 한 답변에서 해결되지 않은 지점에서 출발
2. 이전 질문보다 더 근본적이거나 더 넓은 시각을 요구
3. 한 문장으로 명확하게 서술

### 6. Q&A 노트 저장
`questions/{YYYY-MM-DD}_{질문앞30자}.md` 파일을 Write 도구로 생성합니다:

```markdown
# Q: {질문}

*날짜: {오늘}*
*레벨: L{현재 레벨}*

---

## 답변

{답변 전문}

## 근거 논문
{사용한 논문 목록}

---

## ↓ 다음 질문 (L{다음 레벨})

1. {다음 질문 1}
2. {다음 질문 2}
3. {다음 질문 3}
```

### 7. question-tree.md 업데이트
Edit 도구로 `<!-- L{다음 레벨}_END -->` 위에 3개 질문을 추가합니다:
```
<!-- L{다음레벨}_ITEM -->
- {질문}  `[{오늘 날짜}, from: {질문 앞 30자...}]`
```

### 8. 출력
터미널에 다음을 표시합니다:

```
## 답변
{답변}

---

## 다음으로 탐구할 질문 (L{다음 레벨})

1. {질문 1}
2. {질문 2}
3. {질문 3}

> 이 중 하나를 고르거나, 직접 질문을 입력하세요:
> `/review-ask {질문 1}`
```
