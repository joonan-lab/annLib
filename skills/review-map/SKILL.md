# /review-map

논문 간 관계를 분석해 테마 클러스터와 L2 질문을 생성합니다.
**3편 이상 추가된 후 실행하세요.**

## 실행 절차

### 1. 프로젝트 확인
```bash
python -m annlib.review.project find
```
PROJECT_DIR 을 확인합니다.

### 2. 모든 논문 노트 읽기
papers/ 폴더의 `.md` 파일을 모두 Read 도구로 읽습니다.

### 3. 관계 분석 (핵심 사고 단계)
각 논문 쌍에 대해 다음을 판단합니다:

**관계 유형:**
- 🟢 **지지** (Support): A의 결론을 B가 뒷받침
- 🔴 **반박** (Contradict): A와 B의 결론이 충돌
- 🔵 **확장** (Extend): B가 A의 방법/범위를 확장
- 🟡 **선행** (Precede): A가 B의 이론적 기반
- ⚪ **무관** (Unrelated): 공통 주제 없음

### 4. 테마 클러스터 생성
논문들을 2-5개의 테마 그룹으로 묶습니다.
각 테마마다 themes/{테마-이름}.md 파일을 Write 도구로 생성합니다:

```markdown
# 테마: {테마명}

## 소속 논문
- [[{논문파일명}|{저자 et al. (연도)}]]
- ...

## 테마 요약
{이 테마가 다루는 공통 주제 2-3문장}

## 논문 간 관계
| 논문 A | 관계 | 논문 B | 설명 |
|--------|------|--------|------|
| ... | 🟢 지지 | ... | ... |

## 이 테마에서 아직 답 없는 것
{테마 내 미해결 부분}
```

### 5. L2 질문 생성
논문 간 관계에서 생기는 질문을 **최소 5개** 만듭니다.

좋은 L2 질문의 특징:
- 두 논문 이상을 동시에 참조해야 답할 수 있다
- 단순 사실 확인이 아닌 해석이나 통합이 필요하다
- "왜", "어떻게", "어느 조건에서" 로 시작한다

예시:
- "A(2020)는 X가 증가한다고 했고 B(2022)는 감소한다고 했다 — 어떤 조건 차이가 이 모순을 만드는가?"
- "C, D, E 논문이 모두 Y 방법을 쓰는데, 이 방법이 공통으로 측정하지 못하는 것은?"

### 6. question-tree.md 업데이트
Edit 도구로 `<!-- L2_END -->` 위에 L2 질문들을 추가합니다:
```
<!-- L2_ITEM -->
- {질문}  `[{관련 논문들}, {오늘 날짜}]`
```

### 7. 관계 맵 저장
`themes/relationship-map.md` 를 Write 도구로 생성합니다:

```markdown
# 논문 관계 맵

생성일: {오늘 날짜}

## 전체 관계
| 논문 | 지지 | 반박 | 확장 |
|------|------|------|------|
| ... | ... | ... | ... |

## 테마 클러스터
{테마별 논문 목록}
```

### 8. 완료 메시지
테마 수, L2 질문 수, 발견된 주요 관계(지지/반박)를 요약해서 출력합니다.
그리고 L2 질문 목록을 보여줍니다.
다음 단계로 `/review-ask <L2 질문 중 하나>` 를 권합니다.
