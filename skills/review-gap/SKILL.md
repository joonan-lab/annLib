# /review-gap

문헌의 빈틈과 모순을 분석하고 L4 질문을 생성합니다.
**L2 이상의 질문이 어느 정도 쌓인 후 실행하세요.**

## 실행 절차

### 1. 전체 컨텍스트 로드
Read 도구로 다음을 모두 읽습니다:
- `questions/question-tree.md`
- `papers/*.md` (모든 논문 노트)
- `themes/*.md` (있으면)

### 2. 4가지 분석 수행

#### 2-1. 미답 질문 탐색
question-tree.md 의 L1~L3 질문 중 어떤 논문도 직접 답하지 않는 것을 찾습니다.

#### 2-2. 모순 탐지
서로 다른 결론을 내린 논문 쌍을 찾고, 모순의 원인을 추정합니다:
- 연구 대상 집단 차이?
- 측정 방법 차이?
- 시기나 맥락 차이?
- 진짜 모순 (같은 조건에서 다른 결과)?

#### 2-3. 방법론적 공백
모든 논문이 공통으로 쓰는 방법론이 있다면, 그 방법론이 **측정하지 못하는 것**을 찾습니다.

#### 2-4. 암묵적 가정 탐지
논문들이 명시하지 않고 당연하게 여기는 가정들을 목록화합니다.
각 가정에 대해: "이 가정이 틀렸다면 이 분야의 결론은 어떻게 달라지는가?"

### 3. L4 질문 생성
위 4가지 분석에서 나온 **L4 질문 5~8개**를 만듭니다.

L4 질문의 특징:
- 현재 존재하는 논문들로는 답할 수 없다
- 새로운 데이터, 방법, 또는 이론이 필요하다
- 이 분야의 다음 중요한 연구 방향을 암시한다

### 4. gap-analysis.md 저장
`gap-analysis.md` 를 Write 도구로 생성합니다:

```markdown
# Gap Analysis

*생성일: {오늘 날짜}*
*분석 논문 수: {N}편*

---

## 1. 미답 질문
> 문헌에서 직접 다루지 않는 질문들

{목록}

---

## 2. 모순 및 불일치
> 논문들 간 충돌하는 결과

| 논문 A | 논문 B | 주장 차이 | 추정 원인 |
|--------|--------|-----------|-----------|
| ... | ... | ... | ... |

---

## 3. 방법론적 공백
> 현재 방법론이 포착하지 못하는 것

{목록}

---

## 4. 암묵적 가정
> 검증되지 않은 전제들

| 가정 | 반례 가능성 | 함의 |
|------|-------------|------|
| ... | ... | ... |

---

## L4 질문 — 미래 연구 방향

1. {질문 1}
2. {질문 2}
...
```

### 5. question-tree.md 업데이트
Edit 도구로 `<!-- L4_END -->` 위에 L4 질문들을 추가합니다.

### 6. 완료 메시지
다음을 출력합니다:

```
## 빈틈 분석 완료

### 발견된 것
- 미답 질문: {N}개
- 모순·불일치: {N}쌍
- 방법론적 공백: {N}개
- 암묵적 가정: {N}개

### L4 질문 (미래 연구 방향)
1. {질문 1}
2. {질문 2}
...

> gap-analysis.md 에 전체 분석이 저장되었습니다.
> 
> 이 질문들을 깊이 탐구하려면:
> `/review-ask {L4 질문 중 하나}`
```
