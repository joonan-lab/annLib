# /review-add

논문을 추가하고 L1 질문을 생성합니다.

## 실행 절차

### 1. 입력 확인
ARGUMENTS 를 그대로 사용합니다. 없으면 "추가할 논문의 DOI, 키워드, 또는 PDF 경로를 입력하세요."

### 2. 논문 저장
```bash
python -m annlib.review.add <ARGUMENTS>
```
출력에서 `NOTE_PATH=`, `TITLE=`, `CITATION=`, `ABSTRACT=` 를 읽습니다.

오류가 나면 사용자에게 알리고 중단합니다.

### 3. 노트 읽기
NOTE_PATH 파일을 Read 도구로 읽습니다.

### 4. L1 질문 생성 (핵심)
논문 내용을 바탕으로 다음 5가지 유형의 L1 질문을 **각 1개씩** 생성합니다:

- **핵심 질문**: 이 논문이 직접 답하고자 한 질문은 무엇인가?
- **방법 질문**: 이 방법론의 핵심 가정은 무엇이며, 그것이 틀렸다면 결론이 어떻게 바뀌는가?
- **결과 질문**: 가장 놀라운 결과는 무엇이며, 왜 그 결과가 예상 밖인가?
- **한계 질문**: 저자들이 인정하지 않은 한계나 대안 설명은 무엇인가?
- **연결 질문**: 이 연구가 답하면서 동시에 새로 열어놓은 질문은 무엇인가?

### 5. 노트에 L1 질문 기록
Edit 도구로 NOTE_PATH 파일의 `## L1 질문` 섹션을 채웁니다:

```markdown
## L1 질문

| 유형 | 질문 |
|------|------|
| 핵심 | {핵심 질문} |
| 방법 | {방법 질문} |
| 결과 | {결과 질문} |
| 한계 | {한계 질문} |
| 연결 | {연결 질문} |
```

### 6. question-tree.md 에 추가
```bash
python -m annlib.review.tree append 1 "<논문 제목 축약>"
```
위 명령어에 stdin으로 생성한 질문 5개를 줄바꿈으로 전달합니다.

또는 Edit 도구로 question-tree.md 의 `<!-- L1_END -->` 바로 위에 직접 추가합니다:
```
<!-- L1_ITEM -->
- {질문}  `[{논문 저자 et al. (연도)}, {오늘 날짜}]`
```

### 7. 완료 메시지
```
## ✅ 논문 추가 완료

**{TITLE}** ({CITATION})

**생성된 L1 질문 5개:**
1. {핵심 질문}
2. {방법 질문}
3. {결과 질문}
4. {한계 질문}
5. {연결 질문}

논문이 쌓이면 `/review-map` 으로 관계를 분석하세요.
현재 논문 수: (papers/ 폴더 .md 파일 개수)
```
